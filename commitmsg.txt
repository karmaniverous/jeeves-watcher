feat: memory slot takeover with virtual rules API

Service:
- VirtualRuleStore: in-memory store for externally registered inference rules
- POST /rules/register: accept { source, rules }, merge into pipeline
- DELETE /rules/unregister: remove rules by source (body or :source param)
- POST /points/delete: delete points matching a Qdrant filter
- virtualRules key added to merged virtual document
- onRulesChanged callback recompiles config + virtual rules

- kind: "memory" in manifest (auto-disables memory-core)
- memory_search: lazy init virtual rules, semantic search with domain filter
- memory_get: filesystem read with workspace path validation
- Refactored: watcher tools extracted to watcherTools.ts (300 LOC limit)
