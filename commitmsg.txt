feat: composable array schema merge (concatenate set values)

When two schemas both define a set value for an array-typed property,
the values are concatenated instead of the last schema winning.

This enables composable domain tagging: a base schema contributes
["email"] and a meta schema contributes ["meta"], producing
["email", "meta"] in the merged result.

- ResolvedProperty.set widened to string | unknown[]
- mergeSchemas concatenates array sets instead of replacing
- resolveAndCoerce resolves each array element individually
- extractSetValues return type updated
- 3 new tests: array concat, scalar-into-array, non-array last-wins
